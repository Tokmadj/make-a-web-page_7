Lesson 1: The Basics of the Web and HTML

How The Internet Works

The Internet is the worlds largest network for computers. When someone visits a webpage, their browser sends a HTTP Request to the Server

The Server finds the correct HTML document and sends it back to the users computer where a Browser interprets the page and displays it on the user's computer screen. This video gives a better insight of it.

HTML

HTML stands for Hypertext Markup Language. The HTML documents form the overwhelming majority of the documents on the internet.

There is two aspects to HTML. One is the Text Content which describes "what you are able to see" and the second part is Markup which describes "how it looks like". This video provides a better understanding.

Tags and Elements

Now HTML documents are made of certain things called Elements. When programmers write code, they tell browsers the type of elements to use when writing HTML Tags. This video provides a better understanding of the difference.

Why are Computers stupid

Computers are not really stupid. They are quite smart. The problem lies in that computers take things word for word and interpret things litterally. That is why it is important to double check and make sure a programmer does not make a little error that could transcend to something bigger later.

Inline vs Block

HTML elements are either one of two things. They are either Inline or Block. Inline cuts out the content, while block elements form an "invisible box" around the content.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

HTML elements are either one of two things. They are either Inline or Block. Inline cuts out the content, while block elements form an "invisible box" around the content.

The "Tree-Like" structure of HTML

The fact that elements can have other elements inside them, is what makes HTML into "Tree-Like" structures. The connection between one element and another can go on and on.

On any internet browser these elements appear to be in a series of boxes contained in other boxes

The relationship between indented HTML and boxes

When somone reads an HTML document as text, they will see a wave of indentations going up and down the left side of the document. The idea here is that the more idented an element is, the more deeply nested its corresponding box is.

Text Editors

When writing code, some programmers will use special text editors like, (Scratchpad or Sublime). One of the great things these editors do is that they will automatically generate a closing tag when you write an opening tag.

Lesson 3: Adding CSS Style to your HTML struture

Avoiding Repitition

Avoiding repitition is one of the most important aspects of programming.

Avoiding Errors: When programmers don't avoid repitition they will often have to do the same thing over and over again. This will lead to time being wasted and more importantly a error will occur whenever the programmmer decides to change one thing.

Consistancy: When writing a webpage consistancy is a key thing to keep in mind. When you choose an element you can assign a style to that class. When you learn to to this and combine it with avoiding repitition it will make coding a lot easier for you.

CSS

CSS stands for Cascading Style Sheets. They are the design aspect that allow programmers to give certain characteristics to the HTML elements. This is achieved by giving similar HTML elements the same class name and then going into more detail of what style should apply to that class.
